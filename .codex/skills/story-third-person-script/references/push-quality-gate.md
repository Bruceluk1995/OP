# Third-Person Push Release Gate

Use after the premise is fixed and before surface lint. This is an editorial gate; metrics cannot replace it.

Read `../../story/references/push-prompt-architecture.md`, `../../story/references/push-entertainment-gate.md`, and `../../story/references/push-retention-chain.md` first. Their invested narrator/listener and audience emotional contracts, entertainment map, `1 + 3 + N` plan, adversarial read, and saved-body evidence map are blocking parts of this gate.

## Lock the Oral Spine

```text
abnormal outcome:
ordinary choice/cause behind it:
immediate pressure, status change, danger, live question, or optional premise-supported cost:
wall hook 1 / new pressure:
wall hook 2 / new pressure:
wall hook 3 / new pressure:
what the audience is waiting to understand:
narrator stance and practical contrast:
stable protagonist/supporting labels:
first decisive proof:
main irreversible conflict:
middle mechanism change 1:
middle mechanism change 2:
earlier choice/tool/relationship recycled by the climax:
final action that proves the ending:
```

Open with the abnormal outcome and causal gap. Do not spend the first screen walking into a room, listing characters, or replaying routine dialogue.

## Five Draft Passes

1. **Report pass**: draft modules as `cause -> protagonist/companion choice -> visible result -> new pressure`.
2. **Selection pass**: compare the body with the high-value event and entertainment scans. Expand human conflict/reversal/revelation/relationship/status/choice/payoff, bridge technical state changes and necessary chronology once, and cut the rest.
3. **Narrator-stance pass**: add practical contrast, selective external reaction, and stable sparse labels. Remove name-led chronology and unjustified mind reading.
4. **Change pass**: name the changed fact in every major module. Merge repeated proof, preparation, explanation, travel, mood, and crowd reaction.
5. **Payoff pass**: make the climax reuse an earlier choice, tool, relationship, mistake, or restraint. End on observable changed action or status, then stop explaining.

Draft natural spoken sentences and short oral paragraphs through these passes. Subtitle splitting is the final surface operation, never the drafting method.

## Blocking Tests

- **Opening-gap test**: first screen contains a person, abnormal result, ordinary cause/choice, immediate consequence, and live pressure.
- **Speaker/listener test**: the wording must sound like the locked invested narrator telling one imagined listener why this person's story matters. A synopsis, school essay, institutional report, or database voice fails.
- **Emotional-job test**: every expanded module changes the audience's pressure emotion or desired next action through concrete evidence and character response. Plot movement without emotional movement is bridge material.
- **Opening-sustain test**: after the stop hook, three different pressures carry the opening block to concrete proof. Biography, lore, restated mystery, and crowd surprise do not count.
- **Proof/conflict-timing test**: use the shared position diagnostics. A late first proof or main conflict defaults to structural rewrite unless an exact earlier pressure performs the same function.
- **Neutral-summary test**: deleting narrator judgments and contrasts must materially weaken the piece. If not, rewrite the narrator stance.
- **Stable-label test**: names and role labels remain clear but sparse. Repeated `name + は` or decorative aliases fail.
- **Knowledge-boundary test**: hidden motives and thoughts are not stated as fact through convenient head-hopping.
- **Camera test**: routine movement, greetings, and ordinary dialogue are compressed into consequential reporting.
- **Uniform-depth test**: if routine setup, travel, lore, procedure, and high points receive similar space, return to the high-value event scan. Push narration does not summarize every event fairly.
- **Human-heat test**: every expanded module must contain a present desire, active blocker, visible collision, and human price/status/relationship change. Technical state change alone is a bridge.
- **Flowchart/procedure-run test**: if roughly 200-400 consecutive characters mainly assign tasks, explain routes/rules, or report system operations while nobody chooses, refuses, loses, risks, or changes a relationship, return to selection and structure.
- **High-point rotation test**: if adjacent peaks are larger versions of the same proof, merge one and replace it with a relationship, information, tactic, status, cost, or choice change.
- **Information-value test**: if narrator lines only repeat visible action, add causal interpretation, practical judgment, comparison, hidden implication, or consequence—or cut them.
- **Chronology-run test**: if `then / after that / meanwhile` carries the account without changed pressure, compress the run to one bridge and enter the next high-value event.
- **Production-leak test**: phrases such as `这里不展开 / 为了节省时间 / 后面再说 / 不再赘述 / 同样过程省略` expose the editor. Delete the meta-comment and make the next changed result itself perform the bridge.
- **Narrator-stance module test**: every expanded 200-400-character module must contain deliberate selection, contrast, or consequence framing. If it survives as a neutral event inventory, rewrite narrator stance or compress it.
- **Decisive-voice test**: if a long human conflict is paraphrased entirely into one neutral voice, preserve the few exact lines whose wording changes status, allegiance, humiliation, danger, or choice. Do not turn the result into a transcript.
- **Protagonist-agency test**: the protagonist makes the unusual choice that causes proof, escalation, or climax. Witnessing is insufficient.
- **Middle-change test**: after first proof, change problem, stakes, tactic, relationship, or cost.
- **Climax-debt test**: the solution depends on something established before the climax, not an instant answer or stronger attack alone.
- **Companion-agency test**: named companions materially change plan, information, cost, rescue, or result. Reaction shots do not count.
- **Ending-action test**: show changed behavior, routine, object, relationship, or status; delete the moral explanation after it.
- **Payoff-placement test**: the title/central promise normally pays inside the final 15-25%. If earlier, identify a stronger and different promise that remains alive and later pays.
- **Post-payoff-tail test**: after the final central payoff, default to roughly 10-15% or less for one consequence beat and one ending proof. A new mechanism, policy list, second test/climax, or long safe-return procedure blocks release.
- **Audio-only test**: hide the line breaks and read aloud. Subjects, causality, human conflict, and punch lines must remain understandable without visual subtitle layout.
- **Over-fragmentation test**: a median spoken unit at or below 12 characters with more than 80 units per 1,000 non-whitespace characters is a recombination candidate, not energetic delivery.
- **Connector-stuffing test**: repeated `そこで／しかも／そして／しかし` or equivalents added to create a visible ratio must be removed. There is no connector quota.
- **Hook-honesty test**: hooks arise from changed facts and consequential choices, not forced reversals or “he did not know” filler.

Any failure returns to the earliest responsible pass. Do not patch only one line when a whole module has the wrong function.

## Blind Read

```text
earliest line where interest drops:
what the audience was waiting for:
what the draft supplied instead:
earliest neutral-summary or scene-camera line:
first low-value event expanded too far:
first high-value event compressed too much:
first repeated high-point function:
first production-process leak or none:
first repeated name/label or head-hop:
first repeated conclusion:
first place entertainment dies:
first 200-400-character flowchart/procedure run:
first module without human collision:
first neutral narrator-stance module:
first important character used only as a function/module:
middle mechanism change:
protagonist-caused proof:
companion material action:
climax's recycled debt:
ending's proof action:
title/main promise payoff position:
stronger promise after it or none:
post-final-payoff share:
five memorable lines, choices, or actions after the opening:
audio-only verdict:
verdict: rebuild core / major structural rewrite / scene rewrite / line edit / ready for user read
```

Append the complete saved-body evidence map from the shared retention contract. Only an evidence-supported `ready for user read` proceeds to surface lint. A surface pass proves subtitle shape only.
