# 短篇拆文质量清单

本文件只用于判断“源文为什么有效或失效”，不把拆文变成评分比赛。拆解报告自身的完整性按 `output-templates.md` 检查，语言清理按 `anti-ai-writing.md` 与 `banned-words.md` 检查。

## 先还原读者体验

- 第一屏承诺了什么结果或情绪？
- 读者在每个阶段具体等什么？
- 最早可能退出的位置在哪里？
- 哪个场景真正改变了选择、关系、信息、风险、代价或预期？
- 哪些段落只是流程、说明或重复证明？

## 必须拆出的六个对象

### 1. 故事核

用一句因果句写清：谁想要什么，因为谁或什么受阻，于是采取何种异常行动，最终付出或赢得什么。不能只写题材标签。

### 2. 读者债务

记录开头欠下的疑问、屈辱、危险、欲望与关系债，以及每笔债的偿还位置。没有偿还的债标为失效，不因为结尾热闹而忽略。

### 3. 状态变化

每个主要场景写成：进入状态 → 人物选择 → 退出状态。若退出状态与进入状态相同，判断它是否应删、并或改写。

### 4. 中段机制

识别故事如何避免重复：对手适应、代价升级、目标改变、关系转向、认知翻转或规则失效。若中段只重复同一证明，明确标出。

### 5. 人物选择

区分人物主动选择与作者安排。重点记录哪一个选择让局势不可逆；“被告知、被救、被承认”不能自动算主角行动。

### 6. 叙述声线

第一人称分析知识边界、注意力偏好、判断习惯、误解与隐瞒；第三人称分析镜头选择和叙述距离。不能用代词数量代替声线分析。

## 可借鉴与不可复制

- 可借鉴：承诺功能、因果机制、场景职责、情绪债务、反转条件。
- 不可复制：原句、专名、独特桥段顺序、角色外观和标志性道具组合。
- 输出“为什么有效”与“换壳后仍需重新设计什么”，不提供套模板填空。

## 结论

结论使用 `核心成立 / 局部失效 / 结构失效`，并给出证据位置。不要给总分，也不要用固定字数间隔判断节奏。
