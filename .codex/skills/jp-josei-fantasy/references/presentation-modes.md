# Josei Fantasy Presentation Modes

Before planning or drafting any female-audience fantasy romance, present this menu unless the user already selected a combination:

> 请选择成品形式（可多选，例如 `2+4`）：
> 1. 传统女性向幻想恋爱网文・第一人称
> 2. 传统女性向幻想恋爱网文・第三人称
> 3. 动漫解说／推文口播・第一人称
> 4. 动漫解说／推文口播・第三人称

Allow multiple selections. Produce separate versions by default; never mix viewpoints merely because multiple options were chosen. Traditional mode stays scene-based. Push mode uses the shared retention chain; `push-opening-template-deck.md` is optional ideation only when explicitly requested.

Immediately normalize the reply instead of passing its number downstream: `1 -> presentation=traditional, viewpoint=first`; `2 -> presentation=traditional, viewpoint=third`; `3 -> presentation=push, viewpoint=first`; `4 -> presentation=push, viewpoint=third`. A raw menu code has no meaning outside this menu. Once push is selected, keep it sticky until the user explicitly changes presentation.

For options 3 and 4, read `../../story/references/push-prompt-architecture.md`, `../../story/references/push-entertainment-gate.md`, `../../story/references/push-retention-chain.md`, and `../../story/references/flan-push-strict-mode.md`. Lock a concrete speaker/listener and audience emotional contract before story structure. First-person female push keeps the same recap surface as third person while changing knowledge boundary, information selection, judgment, self-protection, reaction, and sentence flavor. It is never a pronoun swap and never inherits traditional scene depth or dialogue volume.

Do not infer mode or viewpoint from length, YouTube, one-shot, romance subtype, or packaging request.
