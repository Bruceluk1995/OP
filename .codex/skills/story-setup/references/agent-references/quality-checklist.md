# 写作智能体质量合同

这是部署给各写作智能体的共享接口说明，不是共享创作模板。每个智能体仍以自己的工位合同为准；本文件只规定交接时如何判断是否该退回上游。

## 总原则

- 先盲读作品，再看大纲、指标、对标和作者说明。
- 故事核、因果、人物选择或视角失败，不能由字数、格式、文句或其他优点抵消。
- 不输出总分。输出最早失败位置、证据、成因、所属工位和最小返工动作。
- 数字只能描述样本，不能自动判定作品好坏。

## 五项门槛

### 1. 读者承诺

- 开头让读者具体等待一个结果、答案、关系变化或情绪偿还。
- 当前章或短篇的主角欲望与阻力清楚。
- 最早退出点能被明确指出，而不是笼统说“节奏慢”。

### 2. 因果推进

- 结果会成为后续原因。
- 每个主要场景改变选择、关系、信息、风险、代价或读者预期。
- 同一结论证明一次后转向后果、反扑或新问题，不换道具重复演示。

### 3. 人物选择

- 关键结果主要来自人物选择，不靠巧合、突然降临的权威或反派自毁。
- 主角拥有行动权；配角与对手有自己的利益和适应。
- 角色声音来自注意力和判断逻辑，不靠口头禅。

### 4. 叙述成立

- 第一人称遵守知识边界，并有选择性观察、判断、误解或隐瞒。
- 第三人称保持稳定的叙述距离和镜头职责。
- 段落按动作、信息和镜头变化自然切换，不按固定行长制造“快节奏”。

### 5. 偿还与余波

- 开头欠下的主要疑问、羞辱、危险、欲望或关系债得到结果。
- 高潮带来不可忽略的后果，而不是再演示一次能力。
- 结尾呈现新状态，不复述主题或解释读者已经看懂的内容。

## 工位边界

| 失败类型 | 返回工位 |
|---|---|
| 承诺、欲望、阻力、结局因果失效 | 故事架构 |
| 顺序、重复、中段机制失效 | 大纲 / 结构 |
| 场景只有流程、说明、围观 | 正文场景重写 |
| 时间线、设定、人物状态冲突 | 一致性检查 |
| 故事成立但语言僵硬或模板化 | 行文修改 / 去 AI 味 |

一致性智能体只报告事实冲突，不能宣布作品文学质量通过；写作智能体不能用自己的自检代替独立盲读。
