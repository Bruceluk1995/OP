# Isekai One-Shot Blueprint

Use before drafting every standalone male-audience isekai short.

Plan for colloquial, direct prose. Every beat should move through action, dialogue, choice, proof, cost, reward, or consequence; do not reserve space for long scenery, atmosphere, or psychology passages.

`Structure fingerprint` uses short functional tags, not plot prose. Reuse the same tag vocabulary across projects so the ledger can detect macro-structure repetition even when the title, skill, and opening card differ. Example: `modern-worker > exploitative-contract > recruit-outcasts > external-assault > public-audit > legal-recognition > comic-title`.

## One-Shot Rhythm

For 15k one-shots, use 12-18 meaningful beats. Every 2,000-3,000 characters should deliver at least one visible payoff, reversal, cost, public proof, reward, or stronger pressure.

Recommended distribution:

- 0-1,500 chars: opening bomb, protagonist situation, cheat/promise.
- 1,500-4,000 chars: first test and concrete failure/success.
- 4,000-7,000 chars: escalation, enemy/world misread, skill limit or cost.
- 7,000-10,000 chars: midpoint proof and social/public consequence.
- 10,000-13,500 chars: strongest pressure and climax.
- 13,500-16,500 chars: reward, zamaa/status change, emotional closure.

## Subtype Payoff Floors

For trend-driven one-shots, route with `trend-theme-router.md` first. If a referenced `jp-isekai-plan` type pack contains a six-episode structure, compress only its required payoffs into the one-shot rhythm below; do not output a six-episode outline.

- Battle leveling: monster rule, failed first attempt, tactical adjustment, kill/result, drop/rank/skill consequence.
- OP dominance: enemy misread, restrained choice, overwhelming reveal/command, witness reaction, faction/reputation consequence.
- Exile zamaa: old party misunderstands the protagonist's function, the one-shot visibly proves that function, old party suffers concrete loss.
- Dungeon boss: dungeon rule, resource pressure, boss pattern, skill adaptation, reward/drop.
- Academy-game knowledge: route/death flag, changed action, suspicious ripple, public result.
- Tamer: companion behavior problem, bond trigger, evolution/skill, public reaction.
- Earth-commute: modern object/tool/food/medicine creates advantage, secrecy cost, cross-world consequence.
- Dungeon-master/territory: trap/floor/resource/subordinate command changes a battle or political situation.
- Slow-life reward: comfort is allowed only if there is an object payoff, social proof, relationship shift, or quiet danger resolved.

## Cultural Localization Floor

If the source premise comes from Chinese 玄幻、修仙、武侠、朝堂、宫斗、赘婿、龙傲天, identify the original function before writing and rebuild it for Japanese RPG/light-novel fantasy.

- 宗门/门派/师尊/长老 -> 冒険者ギルド、魔法学院、騎士団、教会、研究塔、ギルドマスター、教官.
- 修炼/灵根/境界/飞升 -> スキル習得、魔力適性、ランク昇格、ジョブ進化、ダンジョン踏破.
- 法宝/灵器/飞剑 -> 魔道具、聖剣、杖、魔剣、アーティファクト, with clear RPG function.
- 皇帝/朝堂/圣旨/夺嫡 -> 王国、王命、王位継承争い、貴族派閥、騎士団会議.
- 后宫/冷宫/嫡庶 -> 王宮派閥、離宮幽閉、相続順位、異母兄弟 rivalry; avoid palace-harem logic unless requested.
- 赘婿打脸/家族羞辱 -> 追放パーティー、没落貴族、商会契約、領地経営、ギルド査定, with visible proof and reward.
- 龙傲天无敌 -> OP reveal through skill, command, dungeon rule, monster ecology, or faction consequence, not abstract destiny talk.

## Ending Rule

Close the central promise:

- enemy defeated or publicly corrected;
- skill/cheat proven;
- reward gained;
- protagonist's next life direction chosen;
- emotional/social status changed.

Do not end with an unresolved "next episode" object. If a final object appears, it should symbolize the completed change, not withhold the main payoff.
