# Type Pack: Exile Zamaa / Support-Class Reversal

Use for 追放ざまぁ, expelled party member, healer, appraiser, porter, rear guard, buffer, debuffer, repairer, cook, mapmaker, scout, storage, or any "useless support was actually essential" engine.

## Benchmark Protocol

Use hot works only as functional references: invisible contribution, old party collapse, public proof, clean zamaa. If available, read `对标/exile-support-reversal/{work}/拆文报告.md`.

## Core Engine

- The protagonist's value was invisible because it prevented problems before they appeared.
- After exile, the old party loses logistics, buffs, repair, scouting, storage, healing, cooking, mapping, negotiation, or boss-mechanic coverage.
- The protagonist should not obsess over revenge. They solve visible problems elsewhere; the old party collapses by cause and effect.

## Six-Episode Season

1. Exile and first proof: kicked out as "useless"; solves a practical combat/logistics problem for new witnesses.
2. Old party crack: the former team fails a simple quest because one invisible function is gone.
3. New role demand: guild, merchant, village, or party pays for the protagonist's support function.
4. Public comparison: same dungeon/monster/problem exposes the gap between old party and protagonist.
5. Zamaa escalation: old party blames, steals, or challenges; their misunderstanding becomes public.
6. Clean break: protagonist gains rank/base/partner; old party loses trust, contract, money, or status.

## Required Payoffs

- What the old party misunderstood.
- Visible support function.
- Old-party failure with cause.
- New witness reward.
- Public zamaa consequence.

## Avoid

- Making the old party stupid without mechanics.
- Revenge speeches replacing proof.
- Support skill becoming generic attack magic without identity.
