# Type Pack: Battle Leveling / Dungeon Grind

Use for 打怪升级, dungeon grind, boss drops, rank growth, level-up, skill variants, adventurer guild, monster combat, and "いずれ主人公最強".

## Benchmark Protocol

Use hot works only as structural references: floor rules, monster weakness, skill test, drop economy, rank-up rhythm, boss gate. If available, read `对标/battle-leveling/{work}/拆文报告.md`.

## Core Engine

- The protagonist starts low, underestimated, or under-equipped.
- Each episode must contain a real combat or monster problem.
- Growth must be visible: rank, drop, skill variant, stat, party role, guild access, map access, or boss qualification.
- The protagonist wins through a tactical read, not just louder power.

## Six-Episode Season

1. First kill above assumed rank: weak-looking skill solves one monster rule; earns a small drop.
2. Guild proof: skeptical staff/adventurer sees the result; protagonist receives a low-rank dungeon permit.
3. Skill combination: first failure forces an adjustment; new skill variant or combo appears.
4. Party role proof: protagonist's skill changes a team fight; former doubter reacts.
5. Boss gate: drop/key/rank grants access; rival party or monster ecology creates pressure.
6. First boss clear: visible win, rare drop, rank-up, and a stronger dungeon/faction hook.

## Required Payoffs

- Monster rule.
- Failed or risky first use.
- Tactical adjustment.
- Kill/clear result.
- Drop/rank/level/skill/status consequence.

## Avoid

- Replacing combat payoff with only shopkeeping, herbs, travel, or comfort.
- Pure status windows without scene-level action.
- Fights that are only "he swung, it died" with no rule.
