# First-Person Push Benchmark

Use this benchmark for anime recap/push narration. It is a synthetic teaching example, not source text to imitate.

## Reject: First-Person Novel Scene

```text
俺は工房の扉を開けた。
中では三人の兵士が壊れた盾を囲んでいる。
「直せるのか」
隊長が不安そうに聞いた。
俺は盾へ手を伸ばした。
```

This acts out the room in real time. The narrator is a camera; dialogue carries ordinary setup; the abnormal result and causal gap are delayed.

## Target: First-Person Push Account

```text
俺は壊れた盾を一枚直しただけで
翌朝には国境砦の隊長にされていた
しかも修理に使ったのは伝説の鉱石ではない
食堂で捨てられる鍋底の鉄くず
問題は盾が強くなりすぎて
敵の攻撃より先に砦の門を壊したことだった
```

The opening gives the abnormal outcome, ordinary cause, contrast, visible proof, and new problem. Each line adds a fact or causal turn. The narrator selects details through practical disbelief.

## Narrator-Mouth Test

Before drafting, answer:

```text
what I notice first:
what I call ridiculous or dangerous:
how I judge money/status/risk:
what fact I would omit as obvious:
what misunderstanding is mine:
what short reaction only I would say:
```

## Conversion Rules

- Move the strongest outcome or contradiction to the opening.
- Report routine movement and keep only choices with consequences.
- Convert most dialogue into attributed or implied narration; preserve exact quotation only when wording changes the relationship, decision, humiliation, or payoff.
- Replace neutral chronology with cause -> result -> complication.
- Keep brief self-reaction; remove diary explanation and generic jokes.
- End a module after something changes, while the next pressure is still live.
- If the same passage still works after replacing `俺` with a character name, the narrator mouth is too weak.
