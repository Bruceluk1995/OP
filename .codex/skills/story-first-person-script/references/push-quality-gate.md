# First-Person Push Release Gate

Use this after the premise is fixed and before surface lint. This is an editorial release gate for narrator-led anime recap/push copy. Metrics cannot replace it.

Read `../../story/references/push-prompt-architecture.md`, `../../story/references/push-entertainment-gate.md`, and `../../story/references/push-retention-chain.md` first. Their speaker/listener and audience emotional contracts, entertainment map, `1 + 3 + N` plan, adversarial read, and saved-body evidence map are blocking parts of this gate.

## Lock the Oral Spine

Write these compact answers before drafting:

```text
abnormal outcome:
ordinary choice/cause behind it:
immediate cost or status loss:
wall hook 1 / new pressure:
wall hook 2 / new pressure:
wall hook 3 / new pressure:
what the audience is waiting to understand:
narrator's professional/lived filter:
first decisive proof:
main irreversible conflict:
middle mechanism change 1:
middle mechanism change 2:
earlier choice/tool/relationship recycled by the climax:
final action that proves the ending:
```

The opening must expose the abnormal outcome and causal gap. Do not spend the first screen walking into a room, introducing a party, or replaying routine dialogue.

## Five Draft Passes

1. **Report pass**: draft each major module as `cause -> choice -> visible result -> new pressure`. Compress routine movement and setup.
2. **Selection pass**: compare the body with the high-value event and entertainment scans. Expand human conflict/reversal/revelation/relationship/status/choice/payoff, bridge technical state changes and necessary chronology once, and cut the rest.
3. **Narrator-mouth pass**: let the narrator select concrete details through work, class, money, danger, shame, desire, or a recurring practical bias. Add judgment at decision points, not as a joke quota.
4. **Change pass**: name the changed fact in every major module. Merge or replace consecutive modules that only repeat proof, preparation, explanation, travel, or mood.
5. **Payoff pass**: make the climax reuse an earlier choice, tool, relationship, mistake, or restraint. End on an observable action or changed social behavior, then stop explaining.

Draft natural spoken sentences and short oral paragraphs through these passes. Subtitle splitting is the final surface operation, never the drafting method.

## Blocking Tests

- **Opening gap**: the first screen contains a person, abnormal result, ordinary cause/choice, and an unanswered pressure. If it only promises future excitement, rebuild it.
- **Speaker/listener test**: the wording must sound like the locked narrator speaking to one imagined listener for a present reason. A synopsis, school essay, institutional report, or generic creator voice fails.
- **Emotional-job test**: every expanded module changes the audience's pressure emotion or desired next action through concrete evidence and protagonist response. Plot movement without emotional movement is bridge material.
- **Opening sustain test**: after the stop hook, three different pressures must carry the opening block to concrete proof. Biography, lore, and restated mystery do not count.
- **Proof/conflict timing test**: use the shared position diagnostics. A late first proof or main conflict defaults to structural rewrite unless an exact earlier pressure performs the same function.
- **Pronoun-swap test**: if replacing `俺／僕／私` with a name leaves the account equally convincing, the narrator mouth is too weak. Rewrite information selection and judgment, not pronouns.
- **Camera test**: if routine doors, walking, greetings, and dialogue are acted out in real time, convert the scene to consequence-led reporting.
- **Uniform-depth test**: if routine setup, travel, lore, procedure, and high points receive similar space, return to the high-value event scan. Push narration does not summarize every event fairly.
- **Human-heat test**: every expanded module must contain a present desire, active blocker, visible collision, and human price/status/relationship change. Technical state change alone is a bridge.
- **Flowchart/procedure-run test**: if roughly 200-400 consecutive characters mainly assign tasks, explain routes/rules, or report system operations while nobody chooses, refuses, loses, risks, or changes a relationship, return to selection and structure.
- **High-point rotation test**: if adjacent peaks are both larger versions of the same proof, merge one and replace it with a relationship, information, tactic, status, cost, or choice change.
- **Information-value test**: if narrator lines only describe visible action, add causal interpretation, practical judgment, comparison, hidden implication, or consequence—or cut them.
- **Chronology-run test**: if the account moves through `then / after that / meanwhile` without changed pressure, compress the run to one bridge and enter the next high-value event.
- **Production-leak test**: phrases such as `这里不展开 / 为了节省时间 / 后面再说 / 不再赘述 / 同样过程省略` expose the editor instead of the character. Delete the meta-comment and make the next changed result itself perform the bridge.
- **Module voice-swap test**: replace the narrator with a neutral camera inside each expanded 200-400-character module. If the module loses no useful selection, judgment, misunderstanding, refusal, or choice, rewrite the module's narrator mouth.
- **Decisive-voice test**: if a long human conflict is paraphrased entirely into one neutral voice, preserve the few exact lines whose wording changes status, allegiance, humiliation, danger, or choice. Do not turn the result into a transcript.
- **Lecture trap**: if six consecutive lines stay on one correction, question-and-answer exchange, practice attempt, or procedure without a new threat, choice, reveal, or result, compress from the decisive judgment to the proof.
- **Result-latency test**: after a decisive instruction or diagnosis, visible proof normally arrives within 2-4 spoken lines. A longer gap requires a distinct live pressure in every intervening line.
- **Explanation-share test**: in a narrative scene, professional explanation cannot occupy more text than visible proof plus immediate consequence. The narrator's profession is a selection filter, not permission to teach the audience the whole lesson.
- **Repeated-lesson test**: only one flagship learning process may establish a repeated upgrade engine. Later upgrades must compress the pedagogy and change problem, tactic, stake, relationship, or consequence.
- **Middle-change test**: after the first decisive proof, the story must change problem, stakes, tactic, relationship, or cost. Another demonstration of the same conclusion is not progression.
- **Climax-debt test**: the solution must depend on something established before the climax. A newly disclosed answer, instant skill readout, or stronger final attack is insufficient by itself.
- **Companion-agency test**: named companions must change the plan, cost, rescue, information, or final result. Agreement, admiration, and surprise do not count.
- **Ending-action test**: show the changed person, routine, object, status, or relationship. Delete the moral paragraph that merely explains the action.
- **Payoff-placement test**: the title/central promise normally pays inside the final 15-25%. If earlier, identify a stronger and different promise that remains alive and later pays.
- **Post-payoff-tail test**: after the final central payoff, default to roughly 10-15% or less for one consequence beat and one ending proof. A new mechanism, policy list, second test/climax, or long safe-return procedure blocks release.
- **Audio-only test**: hide the line breaks and read aloud. Subjects, causality, human conflict, and punch lines must remain understandable without visual subtitle layout.
- **Over-fragmentation test**: a median spoken unit at or below 12 characters with more than 80 units per 1,000 non-whitespace characters is a recombination candidate, not energetic delivery.
- **Connector-stuffing test**: repeated `そこで／しかも／そして／しかし` or equivalents added to create a visible ratio must be removed. There is no connector quota.
- **Hook-honesty test**: hooks come from changed facts and live choices. Generic lines such as “but he did not know” or a forced reversal every three lines do not count.

Any blocking failure sends the draft back to the earliest responsible pass. Do not patch only the failing sentence when the module's function is wrong.

## Cadence Without Formula

- Use short oral lines because each carries one useful turn, not because a fixed line count demands a hook.
- Alternate result, cause, contrast, practical judgment, consequence, and live question. Do not repeat one contrast template throughout the story.
- Keep exact dialogue only when its wording changes humiliation, allegiance, choice, or payoff.
- Use slogans at most as compression after the story has proved them. A slogan cannot perform the proof.
- Let major modules vary in length. Human suspense is uneven.
- Compose complete oral thoughts first and split for subtitles last. Do not break every dependent clause into a separate line.

## Blind Read

Read the body before metrics and answer:

```text
earliest line where interest drops:
what the audience was waiting for there:
what the draft supplied instead:
earliest line where the narrator becomes a camera or author:
first production-process leak or none:
first low-value event expanded too far:
first high-value event compressed too much:
first repeated high-point function:
first repeated conclusion:
first place entertainment dies:
first 200-400-character flowchart/procedure run:
first module without human collision:
first module that fails the voice-swap test:
first important character used only as a function/module:
longest same-action explanation/lesson run:
decisive correction -> visible proof latency:
proof -> external consequence latency:
later repetition of flagship process or none:
middle mechanism change:
climax's recycled debt:
ending's proof action:
title/main promise payoff position:
stronger promise after it or none:
post-final-payoff share:
five memorable lines, choices, or actions after the opening:
audio-only verdict:
ending branch honored: closed / serialized
verdict: rebuild core / major structural rewrite / scene rewrite / line edit / ready for user read
```

Append the complete saved-body evidence map from the shared retention contract. Only an evidence-supported `ready for user read` may proceed to surface lint. A surface pass still proves subtitle shape only.
